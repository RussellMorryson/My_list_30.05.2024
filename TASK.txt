Тестовое задание на тимлида бэкенда проекта второго уровня.
Прислать до 23:55 мск воскресенье 2 июня 2024.

Необходимо создать API бэкенд простого приложения для списка дел.

Функционал приложения:
- создание задачи с обязательными полями "Заголовок", "Описание", 
необязательными полями "Срок", "Теги" (выбираются из списка или 
добавляются новые), автоматически заполняемыми полями "Выполнено", 
"Дата создания",
- редактирование задачи,
- удаление задачи,
- перевод в выполненные,
- просмотр всех задач с фильтром по тегам, статусу выполнения, 
сортировкой по сроку, дате создания, заголовку и пагинацией.

Использовать Django/DRF или FastAPI, postgres, Swagger. 
Будет плюсом использование poetry для зависимостей.

Проект должен разворачиваться одной командой через docker-compose 
и содержать подробное описание в README.

Рабочий код должен находиться в ветке development и содержать 
не менее двух коммитов с описанием.

В качестве ответа прислать ссылку на git-репозиторий и скриншот 
ER-диаграммы базы данных.
